Configurar o IBM Quantum Platform para uma organização

Siga estes passos para configurar o IBM Quantum Platform.

1. Configurar as definições do IAM​

Primeiro, configure as seguintes definições na conta IAM do administrador. Para rever e configurar essas definições, acesse Manage → IAM → Settings.

  • User list visibility determina se os usuários podem ver uns aos outros, independentemente da atribuição de projetos. A configuração enabled restringe a visibilidade dos usuários. Ou seja, os usuários na sua conta não conseguem ver uns aos outros, mesmo que tenham acesso aos mesmos recursos. Escolha o valor adequado para o seu ambiente. Consulte Controlling user visibility para mais informações.
  • API key creation controla se os usuários podem criar chaves de API. No IBM Quantum Platform, é comum usar chaves de API. Se as chaves de API estiverem sendo utilizadas, escolha disabled. Como alternativa, você pode conceder permissões específicas a cada usuário.

2. (Opcional) Criar grupos de recursos​

Pule esta etapa se você estiver usando grupos de acesso que referenciam diretamente instâncias do serviço IBM Quantum.

Se você optou por usar grupos de recursos, acesse Manage → Account → Resource groups (in Account resources) e clique em Create.

3. Criar instâncias do serviço IBM Quantum​

Se você já criou instâncias do serviço IBM Quantum, pule esta etapa.

  • Se você estiver usando grupos de recursos, certifique-se de criar as instâncias de serviço no grupo de recursos adequado.
  • O nome da instância de serviço, como QR-ml, é necessário para referências em grupos de acesso.

Siga os passos em Set up to use IBM Cloud para obter instruções.

4. Criar grupos de acesso para projetos​

Primeiro, crie uma função personalizada que permita aos usuários realizar ações para trabalhar com instâncias do serviço IBM Quantum. Em seguida, crie um grupo de acesso para cada projeto e conceda a esse grupo o conjunto mínimo de permissões necessárias para trabalhar com os recursos do projeto.

5. Configurar seu provedor de identidade e atribuir usuários​

Siga os passos no tópico adequado, de acordo com o provedor de identidade escolhido:

  • Use o IBM Cloud® como provedor de identidade para usuários que possuem contas IBM Cloud
  • Use o IBM Cloud como provedor de identidade para usuários que não possuem contas IBM Cloud
  • Use um provedor de identidade diferente do IBM Cloud para usuários que não possuem contas IBM Cloud
  • Convide usuários e gerencie seus acessos

6. Configurar seu firewall​

Para configurar seu firewall e permitir o acesso aos endpoints da API IBM Quantum, adicione as seguintes URLs à sua lista de permissões:

  • Serviços globais do IBM Cloud: 
    cloud.ibm.com
    iam.cloud.ibm.com
    api.global-search-tagging.cloud.ibm.com
    globalcatalog.cloud.ibm.com
    resource-controller.cloud.ibm.com
  • Serviços IBM Quantum Qiskit Runtime - região us-east: 
    quantum.cloud.ibm.com
    s3.us-east.cloud-object-storage.appdomain.cloud
  • Serviços IBM Quantum Qiskit Runtime - região eu-de: 
    eu-de.quantum.cloud.ibm.com
    s3.direct.eu-de.cloud-object-storage.appdomain.cloud
  • Serviço de prévia do IBM Quantum Qiskit Functions: 
    qiskit-serverless.quantum.ibm.com
